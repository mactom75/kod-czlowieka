# 🧠 Platforma software — Kod Człowieka

## Human Code Project — Software Platform

Platforma software jest drugim filarem projektu **Kod Człowieka (Human Code Project)**.

Pierwszym filarem jest fizyczna platforma sensoryczna, opisana w `PROTOTYPE.md`.

Drugim filarem jest system software, który pozwala:

- logować się użytkownikom,
- zarządzać urządzeniami,
- prowadzić ciągłość danych przez kolejne kohorty życia,
- analizować dane z interakcji,
- udostępniać bezpieczne podsumowania rodzicom i opiekunom,
- tworzyć anonimowe reprezentacje badawcze,
- udostępniać dane naukowcom w kontrolowany sposób,
- wspierać rozwój modeli Embodied AI,
- łączyć dane z psychologii, psychiatrii, pedagogiki, neuronauki, robotyki i uczenia maszynowego.

Platforma nie powinna być zwykłą aplikacją do podglądu kamery.

Jej celem jest stworzenie bezpiecznej, długoterminowej infrastruktury danych o rozwoju człowieka i interakcji człowiek–technologia–środowisko.

---

## 1. Cel platformy software

Celem platformy software jest połączenie danych z urządzeń sensorycznych z długoterminowym modelem rozwoju człowieka.

Platforma powinna odpowiadać na pytania:

- co dzieje się z danymi po ich zebraniu?
- jak użytkownik widzi historię interakcji?
- jak rodzic lub opiekun otrzymuje użyteczne informacje?
- jak dane przechodzą między kohortami życia?
- jak naukowiec może analizować dane bez naruszania prywatności?
- jak system może wspierać rozwój Embodied AI?
- jak łączyć dane z różnych dziedzin nauki?
- jak uniknąć inwigilacji, scoringu i komercyjnego profilowania?

---

## 2. Główna zasada

Najważniejsza zasada platformy:

> **Platforma nie przechowuje prywatnego życia człowieka.  
> Platforma przechowuje bezpieczne reprezentacje doświadczenia.**

Oznacza to, że system powinien ograniczać dostęp do danych surowych, takich jak:

- pełne nagrania wideo,
- pełne nagrania audio,
- prywatne rozmowy,
- dane identyfikujące rodzinę,
- dane biometryczne bez jasnej podstawy,
- dane wrażliwe bez osobnej zgody i kontroli.

Główną wartością platformy są dane przetworzone:

- zdarzenia,
- wzorce,
- wektory cech,
- trendy,
- kategorie interakcji,
- zmiany w czasie,
- anonimowe profile rozwojowe,
- statystyki kohortowe.

---

## 3. Konto użytkownika

Platforma powinna umożliwiać utworzenie konta użytkownika.

Typy kont:

- rodzic,
- opiekun,
- dorosły uczestnik,
- badacz,
- administrator projektu,
- partner technologiczny,
- instytucja badawcza.

Każdy typ konta powinien mieć inny poziom dostępu.

Rodzic nie powinien widzieć panelu badacza.

Partner technologiczny nie powinien widzieć prywatnych danych rodziny.

Badacz nie powinien widzieć danych identyfikujących osoby, jeśli nie ma do tego osobnej zgody i podstawy prawnej.

---

## 4. Profil uczestnika

W systemie może istnieć profil uczestnika.

Profil uczestnika nie powinien być publicznym profilem społecznościowym.

Jest to prywatny, chroniony profil rozwojowy.

Może zawierać:

- identyfikator uczestnika,
- kohortę wiekową,
- przypisane urządzenia,
- zgody rodzica lub uczestnika,
- historię zdarzeń,
- historię interakcji,
- dane środowiskowe,
- dane z urządzeń,
- podsumowania okresowe,
- status prywatności.

W przypadku dzieci profil powinien być zarządzany przez rodzica lub opiekuna prawnego.
